## Conventions

- **Migration tool:** `Alembic` (async via `async_engine_from_config`)
- **Naming:** `snake_case` for tables and columns, plural table names (`projects`, `tasks`)
- **Primary key:** `BigInteger` autoincrement (`GENERATED BY DEFAULT AS IDENTITY`) — UUIDs intentionally avoided (see [decisions.md](decisions.md), 2026-05-04)
- **Timestamps:** `created_at timestamptz NOT NULL DEFAULT now()`, `updated_at timestamptz NOT NULL DEFAULT now()` — router bumps `updated_at` manually on PATCH (server_default only fires on INSERT). Could move to a PG trigger later if more write paths appear.
- **Soft delete:** no — use audit history (`tasks_history`) for traceability instead
- **Audit:** `tasks_history` is populated by PG trigger; application code does NOT insert there

### tasks_history
**Purpose:** Audit sink — populated by `tasks_audit_trg` on UPDATE / DELETE. Application code does NOT write here.

| column     | type        | constraints                                | notes |
|------------|-------------|--------------------------------------------|-------|
| id         | bigint      | PK, identity                               |       |
| task_id    | bigint      | NOT NULL                                   | NOT a FK — survives task deletion |
| operation  | char(1)     | NOT NULL, CHECK `operation IN ('U','D')`   |       |
| changed_at | timestamptz | NOT NULL DEFAULT `now()`                   |       |
| snapshot   | jsonb       | NOT NULL                                   | `to_jsonb(OLD)` from trigger |

**Indexes:** `ix_tasks_history_task_id`, `ix_tasks_history_changed_at`

### Audit trigger

```sql
CREATE OR REPLACE FUNCTION tasks_audit_fn() RETURNS trigger AS $$
DECLARE
    op_code char(1);
    row_id  bigint;
BEGIN
    IF (TG_OP = 'UPDATE') THEN
        op_code := 'U';
        row_id  := OLD.id;
    ELSIF (TG_OP = 'DELETE') THEN
        op_code := 'D';
        row_id  := OLD.id;
    ELSE
        RETURN NULL;
    END IF;

    INSERT INTO tasks_history (task_id, operation, changed_at, snapshot)
    VALUES (row_id, op_code, now(), to_jsonb(OLD));

    IF (TG_OP = 'DELETE') THEN
        RETURN OLD;
    ELSE
        RETURN NEW;
    END IF;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER tasks_audit_trg
AFTER UPDATE OR DELETE ON tasks
FOR EACH ROW EXECUTE FUNCTION tasks_audit_fn();
```
