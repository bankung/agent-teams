## Conventions

- **Migration tool:** `<Alembic | other>` *(fill in once decided)*
- **Naming:** `snake_case` for tables and columns, plural table names (`users`, `orders`)
- **Primary key:** `id BIGINT GENERATED BY DEFAULT AS IDENTITY` (BigInteger autoincrement) unless a natural key makes sense
- **Timestamps:** `created_at timestamptz NOT NULL DEFAULT now()`, `updated_at timestamptz NOT NULL DEFAULT now()` (with trigger if needed)
- **Soft delete:** uniform `status SMALLINT NOT NULL DEFAULT 1 CHECK (status IN (0, 1))` on every business table (1=active, 0=deleted); app code never issues SQL DELETE
